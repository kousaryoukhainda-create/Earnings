# General Instructions for AI Assistant

## ⚠️ CRITICAL: Before Starting Any Task

### 1. STOP and READ this file first
- Always consult this file before executing any command
- Follow the workflow below for EVERY task

### 2. ASK Before Making Changes
- **DO NOT** modify files without explicit permission
- **DO NOT** run long builds/compiles without asking
- **DO NOT** assume what the user wants - ASK first
- **DO NOT** make multiple changes in sequence without confirmation
- **DO NOT** execute commands that take >30 seconds without warning

### 3. GATHER Information First
Before any action, collect:
- [ ] What is the actual error/crash? (get logs)
- [ ] What has already been tried?
- [ ] What is the expected outcome?
- [ ] What tools/environment are available?
- [ ] User's device setup (phone only? computer connected?)

### 4. MINIMAL Approach
- Start with the **simplest possible solution**
- Make **one change at a time**
- **Verify** before proceeding to next step
- If stuck for >2 steps, **ask for help**

## 📋 Standard Workflow

### For Debug/Crash Issues:
```
1. ASK: "Can you share the crash logs?"
2. ASK: "How are you testing? (phone only / with computer / emulator)"
3. ASK: "What exactly happens when you launch?"
4. ONLY THEN: Analyze and suggest ONE fix
5. WAIT for confirmation before making changes
```

### For Build Issues:
```
1. ASK: "What is the exact error message?"
2. ASK: "Show me your current setup (SDK, Gradle versions)"
3. ASK: "Do you have permission to install SDK components?"
4. Suggest minimal fix, wait for confirmation
```

### For New Features/Changes:
```
1. ASK: "What exactly do you want to achieve?"
2. ASK: "Are there any constraints I should know?"
3. Propose a plan (3-5 steps max)
4. WAIT for approval before starting
5. Execute ONE step, then report back
```

- ❌ Run `./gradlew build` or similar long commands without asking
- ❌ Modify build.gradle.kts without permission
- ❌ Change SDK versions, compile versions, etc. without asking
- ❌ Create/modify configuration files without asking
- ❌ Assume the user has certain tools installed (adb, Android Studio, etc.)
- ❌ Make more than 1 file change without confirmation
- ❌ Run commands that take >30 seconds without warning
- ❌ Make assumptions about the user's environment

### ALWAYS:
- ✅ Ask for crash logs first
- ✅ Ask about user's testing setup (phone only? computer?)
- ✅ Check what already exists before creating
- ✅ Propose before executing
- ✅ Keep changes minimal and reversible
- ✅ Report progress after each step
- ✅ Stop and ask if something unexpected happens
- ✅ Update HISTORY.md after every action
